# Memory

在压缩过程中，可能会导致上下文中重要的信息丢失，这是 agent 使用过程中绝对不允许的，所以这一节就来学习如何将重要的信息进行保存，并在后续需要的时候进行使用。

这里会将特定的内容存储到硬盘上，可以跨 session 保存，类似于人类的长期记忆。

可以将这些需要保存的 md 内容分为 4 种类型：

- user：回答用户是一个怎样的人
- feedback：回答用户让 agent 如何做事
- project：回答这个项目的信息
- reference：回答需要的资源在哪里去查

使用 MEMORY.md 作为索引文件，./memory 下存放具体的记忆内容，将 MEMORY.md 中的内容放在 SYSTEM_PROMPT 中，可以让 LLM 在后续的使用中直接获取想要的内容。

并且在每轮结束后，让 LLM 从聊天信息中提取需要写入到 MEMORY 中的内容。

为了更好地维护整体的记忆内容，还需要在存储了一定数量的内容后进行整理，可以简单设置为当文件数量大于等于 10 个时，触发去重、合并、剪枝的操作。
