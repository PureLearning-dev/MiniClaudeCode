# Task System

s05 中提到了 todo list，其中实现让 Agent 在开始执行之前先进行规划拆解，使执行步骤分明。

但是，在执行时并没有顺序的限制，可能先执行后面的任务，然后又回过头执行前面的任务，导致上下文混乱。

为了解决这样的问题，引入 Task System，实现的核心是实现一个有向无环图，执行一个任务，之前的所有任务都必须完成，并且这个 Task System 会将任务进行落盘。

在实现中，将任务作为一个 json 文件进行落盘，存储在 ./task 目录下。

任务会被引入 3 个状态：

- pending
- in_progress
- completed

一个任务要开始执行，必须要等待它的所有 blockedBy 任务完成，在一个任务完成后，去查找它的可执行的下游任务，然后进行执行，直到整个任务图全部完成。
