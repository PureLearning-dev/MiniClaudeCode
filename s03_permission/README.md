# Permission

虽然 s02 可以做到在 WORKDIR 中进行操作，但是无法防止执行毁灭性的操作，所以在执行操作之前需要判断一下权限。

在日常使用 ClaudeCode 中也是这样的，在执行删除等命令时，cc 会向用户询问是否继续执行，这就是因为删除等操作的权限就是 ask，像读取文件内容命令的权限是 allow，就不需要询问，直接执行就行。

安全不能靠大模型，而要使用严格的代码逻辑进行限制！

在执行命令之前添加一个权限校验模块，其中由三道闸门进行兜底：

1. 拒绝列表
2. 规则匹配
3. 用户审批

如果操作在 1 处捕获，则不允执行，在 2 处捕获，让用户进行审批，如果操作可以通过三道闸门，直接执行即可。

总结来说，一共有三种执行的可能性，拒绝、直接执行、获取用户审批。
